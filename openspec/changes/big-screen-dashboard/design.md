## Context

项目 `admin` 为 Vue 3 + Vite + TypeScript + Element Plus，并已集成 `echarts`，路由由 `admin/src/router.ts` 管理；`App.vue` 目前为“侧边栏 + 顶部栏 + 主内容”布局，并对登录/注册页做了布局分流。

大屏首页的目标是提供一个全屏、信息密度更高的“监控大盘”视图，用于展示系统运行状态与当天触发情况。其信息结构可参考 `大屏.MD` 原型图：

- 顶部：标题、时间、刷新
- KPI：运行中策略、今日触发、推送成功、监控股票数
- 中部：实时触发动态（滚动） + 今日触发趋势（折线图）
- 下部：重点关注股票（列表）
- 底部：最近一次异动详情（迷你K线 + 文本概览）

## Goals / Non-Goals

**Goals:**
- 新增一个独立的大屏路由，并在该路由下使用全屏布局（不显示侧边栏/顶部栏）。
- 首页以“实时监控 + 图表可视化”为核心，刷新频率可配置且不会造成请求风暴。
- 数据口径明确：今日触发统计、推送成功率、趋势图的时间粒度。

**Non-Goals:**
- 不在本次变更中重做现有管理后台导航结构（侧边栏仍保留）。
- 不在本次变更中实现复杂的个股诊断系统与完整的行情详情页（可作为后续迭代）。

## Decisions

### 1) 路由与布局

- 新增路由：推荐使用 `/screen`（简短、语义明确）。
- 布局分流：在 `App.vue` 中新增 `isScreenPage` 判定（`route.path.startsWith('/screen')`），当命中时仅渲染 `<router-view />`，从而获得无导航栏的全屏效果。
- 权限：大屏默认走现有的 `router.beforeEach` 鉴权逻辑（非 public 需要 token）。

### 2) 数据获取策略

大屏数据可通过两种方式获得：

**A. 纯前端聚合（MVP）**
- `/api/strategies`：统计启用策略数、解析 symbols 计算监控股票数
- `/api/trigger-logs`：用 `startDate/endDate` 拉取当天触发（受 `pageSize<=100` 限制）
- `/api/trigger-logs/:id`：拉取最近一条触发的 K 线详情用于迷你图

优点：无需改后端；缺点：统计不准（触发超过 100 条时）、请求次数较多、趋势聚合逻辑分散在前端。

**B. 增加后端聚合接口（推荐）**
- 新增 `GET /api/dashboard/screen`（或 `/api/screen/overview`）返回：
  - `kpis`: 启用策略数、监控股票数、今日触发数、推送成功率/成功数
  - `latestTriggers`: 最近 N 条触发（用于滚动列表）
  - `todayTrend`: 今日按小时（或 15 分钟粒度）聚合的触发次数数组（用于折线图）
  - `watchlist`: 从启用策略的 symbols 聚合出 TopN 股票，并返回最新报价（复用 `fetchStockDataBatch`）
  - `latestTriggerDetailId`: 最近触发 ID（前端据此再请求 `/api/trigger-logs/:id` 取 K 线）

优点：统计准确、一次请求拿齐大部分数据、口径集中；缺点：需要新增后端路由与 SQL 聚合。

本变更建议采用 B，并在前端保留降级：接口不可用时退回 A。

### 3) 可视化方案

- 图表库：使用已集成的 `echarts`。
- 趋势图：折线图/面积图，X 轴为时间（小时或 15 分钟），Y 轴为触发数。
- 迷你 K 线：使用 K 线（candlestick）+ 量能（bar）简化版；优先复用 `/api/trigger-logs/:id` 返回的 `kline` 数据。

### 4) 刷新与性能

- 自动刷新：默认 5~10 秒刷新一次大屏聚合数据；页面不可见（`document.hidden`）时暂停。
- 滚动列表：前端仅维护最近 N 条；当新数据到来时插入顶部并做轻量动效。
- 防抖与并发：同一时间只允许一个刷新请求在途，避免多次点击刷新造成并发堆积。

## Risks / Trade-offs

- **[统计口径与时区]**：`created_at` 字符串比较依赖格式与时区。
  - Mitigation：后端聚合统一以 `substr(created_at,1,10)`（现有用法）并明确使用本地日期；必要时引入服务端日期边界计算。
- **[外部行情接口波动]**：watchlist 报价依赖新浪接口。
  - Mitigation：接口失败时降级为仅展示股票代码与“--”价格，并展示“数据暂不可用”。
- **[大屏布局适配]**：不同分辨率下信息密度与可读性冲突。
  - Mitigation：以 1920x1080 为基准，使用 CSS grid + 自适应字体/间距，并提供缩放模式。
