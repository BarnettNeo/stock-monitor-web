## Context

智能股票量化监控WEB系统（Stock Monitor Web）是一个轻量级的股票监控与推送系统，基于 Node.js 迁移并增加了 Web 管理界面。
项目旨在为用户提供实时的股票监控能力，支持通过不同的策略配置（异动、目标价、技术指标、形态等）对公开行情数据进行扫描，并在触发条件时通过钉钉或企业微信等渠道进行消息推送。系统分为前后端两部分：
- **server**: 基于 Node.js + Express + TypeScript 编写的后端服务，承担 REST API 提供、定时扫描任务（Scheduler）、消息推送、以及通过本地 SQLite (`sql.js`) 的数据持久化工作。
- **admin**: 基于 Vue 3 + Vite + Element Plus 的前端管理后台，提供策略管理、订阅渠道管理及触发日志查看等功能。

## Goals / Non-Goals

**Goals:**
- **统一的前后端分离架构设计**：明确前后端的职责边界，后端负责核心业务逻辑与数据存储，前端负责交互配置与结果展示。
- **可靠的扫描与触发机制**：设计定时轮询策略及冷却机制，确保系统能在指定时间间隔内准确扫描，并且不会在短时间内重复推送相同触发原因的消息。
- **轻量级持久化方案**：通过无需安装原生依赖的 `sql.js` (SQLite WASM) 内存 DB 结合定期持久化到磁盘的方式，实现便捷部署。

**Non-Goals:**
- **不涉及真实交易系统对接**：本系统仅为数据监控和行情分析提醒，不提供实盘交易接口和下单功能。
- **不保证行情数据的绝对实时性与完整性**：由于数据源依赖公开的免费行情接口（如新浪财经），存在请求频率限制或偶发超时的情况，本系统只尽力而为。

## Decisions

- **后端技术栈 (Node.js + Express + TypeScript)**
  - 理由：轻量级，生态丰富，TypeScript 提供了完善的类型推导，降低重构和后期维护成本。
- **前端技术栈 (Vue 3 + Vite + Element Plus)**
  - 理由：Vue3 + Vite 开发体验极佳，构建速度快；Element Plus 提供了开箱即用的后台管理组件库，可以快速搭建管理页面。
- **数据库选型 (sql.js - SQLite WASM)**
  - 理由：免去了原生 SQLite 模块带来的环境依赖和编译烦恼。运行时作为内存数据库提供极高的读写性能，通过定期导出文件或触发式落盘的方式持久化至 `server/data/db.sqlite`，适合轻量级、低并发的单节点应用。
- **指标计算 (technicalindicators 库)**
  - 理由：该库提供了标准的 MACD、RSI、SMA 等技术指标算法，避免重复造轮子，能够可靠地支撑技术指标触发功能。
- **告警推送与冷却机制**
  - 理由：由于扫描任务频率较高（默认 15 秒/次），同一股票在短时间内持续触发相同条件会造成“消息轰炸”。设计 `cooldownMinutes` 冷却机制，根据“策略+股票+原因”的维度记录最近一次推送时间，在冷却期内自动阻断推送。

## Risks / Trade-offs

- **[内存数据库丢失风险]** → Trade-off: 使用 `sql.js` 内存数据库，如果在未落盘前进程意外崩溃，会导致最近几秒的数据丢失。
  - Mitigation: 在任何修改数据的操作（如 API 变更、日志写入）后，触发落盘（或者通过防抖机制批量落盘），以及在进程优雅退出时进行保存。
- **[外部 API 稳定性]** → Risk: 新浪财经公开接口有访问频率限制或不可控的波动。
  - Mitigation: 允许通过 `SCAN_INTERVAL_MS` 调整全局扫描间隔；并在请求失败时捕获异常、记录日志，不阻断下一轮定时任务的执行。
- **[单点性能瓶颈]** → Risk: Node.js 单线程模型在监控成千上万只股票并且高频扫描时，可能遇到 CPU 计算瓶颈。
  - Mitigation: 当前定位为个人或小团队使用的轻量级系统，通常监控股票数量在百级别以内，性能完全满足。若未来需扩展，可重构为多进程或引入消息队列。
